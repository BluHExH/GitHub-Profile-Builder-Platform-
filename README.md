## 📖 Description

**HEX Forge** is a premium, AI-powered web platform that revolutionizes how developers create GitHub documentation and personal branding assets. Instead of spending hours formatting Markdown, crafting profile READMEs, or searching for the perfect badge, HEX Forge does it all in seconds — with AI.

### Why It Exists

Every developer knows the pain of writing a good README. It's tedious, repetitive, and easy to get wrong. Your GitHub profile is your developer resume, yet most profiles are either empty or poorly formatted. Existing tools are fragmented — one for badges, another for README generation, another for stats widgets. HEX Forge unifies everything into a single, beautiful, AI-powered platform.

### What Problems It Solves

- **README Writer's Block** — AI generates production-quality READMEs from just a project name and description
- **Boring GitHub Profiles** — 30+ verified animated GIFs, live stats widgets, and premium templates
- **Badge Hunting** — 500+ shields.io badges across 15 categories in one searchable library
- **Repo Documentation Debt** — Health scores, SEO analysis, and AI improvement plans for any public repo
- **Portfolio Creation** — Build developer portfolios with 4 premium themes and AI content generation
- **Tool Fragmentation** — 12 integrated tools in one dashboard, no account required

### Who It's For

- **Open Source Maintainers** — Generate comprehensive READMEs, contribution guides, and license files
- **Job-Seeking Developers** — Create standout GitHub profiles and portfolios that impress recruiters
- **Indie Hackers & Solo Founders** — Build professional documentation for your SaaS products
- **Technical Writers** — Leverage AI to draft, then refine documentation with the live Markdown editor
- **DevOps & Platform Engineers** — Analyze repository health and get actionable improvement roadmaps

## ✨ Features

### 🤖 AI README Generator

Generate comprehensive, production-quality README files in seconds. Input your project name, description, tech stack, and key features — HEX Forge's AI (powered by Google Gemini 2.5 Flash) produces a complete README with badges, installation guides, usage examples, contribution guidelines, and license sections. Features real-time streaming output, live Markdown preview, and one-click copy/download.

| Feature | Description |
|---|---|

| **Form Validation** | Zod schema validation with React Hook Form |

## 🚀 Installation

```bash
# Clone the repository
git clone https://github.com/hackerhex/hex-forge.git

# Navigate to the project directory
cd hex-forge

# Install dependencies
npm install

# Copy environment variables (optional — app works without them)
cp .env.example .env.local
```

## 🔐 Environment Variables

Copy `.env.example` to `.env.local`. All variables are **optional** — HEX Forge works out of the box without any configuration.

| Variable | Description | Required | Default |
|---|---|---|---|
| `VITE_AI_API_URL` | AI API endpoint URL for streaming generation | No | — |
| `VITE_AI_API_KEY` | API key for the AI service | No | — |
| `VITE_GITHUB_TOKEN` | GitHub personal access token (higher API rate limits) | No | — |
| `VITE_APP_ENV` | Application environment (`development`, `production`) | No | `development` |
| `VITE_DEBUG_MODE` | Enable debug logging | No | `true` |

> **Note:** Without `VITE_AI_API_URL`/`VITE_AI_API_KEY`, AI generation features will show a configuration prompt. All other features (Badge Studio, Markdown Editor, Template Marketplace, Export Center, Team Members, Settings) function fully without AI configuration. Repository Analyzer uses the public GitHub API directly.

## 🎯 Usage

### Getting Started in 60 Seconds

<details>
<summary><b>Step 1: Land on the homepage</b></summary>

Visit the landing page to explore all features. The animated hero section showcases the platform's core value proposition with a typewriter effect. Scroll through features, see how it works, read testimonials, and find answers in the FAQ section.

</details>

<details>
<summary><b>Step 2: Start from the Dashboard</b></summary>

Click "Start building" or navigate to `/dashboard`. The dashboard shows:
- **Stats Cards** — Total projects, READMEs, profiles, analyses with animated CountUp
- **Activity Chart** — 7-day project creation activity with recharts area chart
- **Quick Start** — One-click access to the most popular tools
- **AI Tools Grid** — Grid of all 7 tools with icons
- **Recent Projects** — Last 6 projects sorted by update date with type badges

</details>

<details>
<summary><b>Step 3: Generate a README with AI</b></summary>

Navigate to **README Generator** (`/readme-generator`):
1. Fill in the form: Project Name, Description, Tech Stack, Key Features
2. Optionally add Team/Contributors from your saved roster
3. Optionally add additional context (license, target audience, etc.)
4. Click "Generate README" — watch the AI stream content in real-time
5. Use the Preview tab to see rendered Markdown, Raw tab for source
6. Copy to clipboard, download as `.md`, or save to your project library

</details>

<details>
<summary><b>Step 4: Build a GitHub Profile</b></summary>

Navigate to **Profile Builder** (`/profile-builder`):
1. Choose a visual style (Futuristic, Modern, Creative, or Minimal)
2. Fill in identity, tech, and extras tabs
3. Use the right panel to browse templates, snippets, or GIFs
4. Insert templates to pre-fill form fields, snippets/GIFs to append to output
5. Click "Build Profile" — the AI generates a complete profile README
6. Preview in 4 tabs: Preview (rendered), Stream (live), Raw MD (editable), Rendered

</details>

<details>
<summary><b>Step 5: Analyze a Repository</b></summary>

Navigate to **Repository Analyzer** (`/repo-analyzer`):
1. Enter a GitHub URL (e.g., `https://github.com/facebook/react`) or shorthand (`facebook/react`)
2. Click "Analyze" — the tool fetches live data from the GitHub API
3. View the **Score Breakdown** tab with 5 dimension cards:
   - Animated SVG score ring
   - Progress bar
   - Individual checkmarks/warnings with actionable tips
4. Switch to the **AI Analysis** tab for a comprehensive analysis with Health Summary, Top 5 Priority Actions, README Improvement Plan, Security Recommendations, Growth Strategy, and 3-Month Roadmap

</details>

<details>
<summary><b>Step 6: Create a Badge Collection</b></summary>

## 🧠 AI Workflow

<p align="center">
  <img src="https://user-images.githubusercontent.com/74038190/216649426-0c2ee152-84d8-4707-85c4-27a378d2f78a.gif" width="400" alt="AI Workflow"/>
</p>

HEX Forge's AI system is designed for **real-time streaming** with **automatic resilience**:

```
User Input Form → Zod Validation → Build Prompt → SSE Stream → AI Token Generation → Real-time Rendering
                                                                     ↓
                                                            Auto-retry on 503 (3x)
                                                            Rate-limit cooldown UI
```

### How AI Generation Works

1. **Prompt Construction** — Each page builds a structured prompt combining user input with system instructions:
   - **README Generator**: Project metadata + expert technical writer system prompt
   - **Profile Builder**: User details + style configuration + verified GIF URLs
   - **Portfolio Builder**: Professional info + theme selection + section instructions
   - **Repository Analyzer**: Live GitHub data + health scores + analysis instructions

2. **Streaming Request** — The `sendStreamRequest` function in `src/lib/sse.ts` sends a POST request to the AI endpoint via SSE:
   - Builds a streaming connection with AbortSignal support
   - Includes optional Bearer token authentication
   - Handles streaming response with the `eventsource-parser` library

3. **Response Parsing** — Each SSE event is parsed for text chunks:
   - Chunks are appended to state as they arrive
   - The `Streamdown` component renders incomplete Markdown in real-time
   - Users see content appear character-by-character

4. **Resilience & Error Handling**:
   - **503 Retry**: Up to 2 automatic retries with exponential backoff (3s, 6s)
   - **Rate Limiting**: 429 responses trigger a cooldown UI with animated ring countdown
   - **Network Errors**: Retry with delay, then user-friendly error message
   - **User Cancellation**: AbortController cancels the stream mid-generation

5. **Content Persistence**: Generated content can be saved to localStorage (no backend required) for later editing or export.

<p align="center">

- **Efficient State**: No global state library — local component state and `localStorage` for persistence keeps bundles minimal
- **Animation Performance**: CSS keyframe animations for non-UI animations, Motion for UI animations with hardware acceleration
- **Image Loading**: All GIF assets use `loading="lazy"`, badge images are loaded on-demand
- **Reduced Motion**: Full `prefers-reduced-motion` support — all animations respect accessibility preferences
- **Minimal Dependencies**: Carefully selected libraries, no unnecessary bloat

<p align="center">

- **Content Ownership**: Any content you generate is 100% yours. No claims to ownership.

## ❓ FAQ

<details>
<summary><b>Is HEX Forge free?</b></summary>
Yes — all core features are completely free. No account required. You can generate READMEs, build profiles, browse badges, analyze repos, edit Markdown, and export content without any payment. An optional account lets you save projects across sessions.

</details>

<details>
<summary><b>What AI model powers HEX Forge?</b></summary>
HEX Forge uses Google Gemini 2.5 Flash, one of the fastest and most capable models for developer documentation tasks. The AI integration supports any OpenAI-compatible streaming API — you can configure your own endpoint if desired.

</details>

<details>
<summary><b>Can I use generated content commercially?</b></summary>
Absolutely. All content you generate is yours to use however you like, including for commercial projects, open source repositories, or personal portfolios. No attribution required.

</details>

<details>
<summary><b>How does the Repository Analyzer work?</b></summary>
It fetches live data from the GitHub API (stars, forks, issues, topics, license, etc.), computes health scores across 5 dimensions, then passes the metadata to Gemini AI to generate a comprehensive improvement report with actionable recommendations.

</details>

<details>
<summary><b>Is dark mode supported?</b></summary>
Yes — HEX Forge fully supports light and dark mode across every page. Your preference is saved automatically in `localStorage` and respects your system preference on first visit.

</details>

<details>
<summary><b>Do I need to create an account?</b></summary>
No. All features work without an account using `localStorage` for persistence. An account (optional) enables cloud sync and multi-device access.

</details>

<details>
<summary><b>How do I configure the AI?</b></summary>
Copy `.env.example` to `.env.local` and set `VITE_AI_API_URL` and `VITE_AI_API_KEY` with your AI provider's endpoint and key. Without configuration, AI features will show a setup prompt.

</details>

<details>
<summary><b>Where is my data stored?</b></summary>
All your projects, team members, and settings are stored in your browser's `localStorage`. No data is sent to any server unless you explicitly configure an AI API endpoint. Clearing your browser data will remove your projects — export important work before clearing.

</details>

<details>
<summary><b>What browsers are supported?</b></summary>
All modern browsers (Chrome 90+, Firefox 90+, Safari 15+, Edge 90+). IE11 is not supported.

</details>

<details>
<summary><b>How do I report a bug or suggest a feature?</b></summary>
Open an issue on GitHub or reach out via the contact links below. Feature requests and bug reports are always welcome.

</details>

### Development Guidelines

| Area | Guideline |
|---|---|
| **Code Style** | Follow existing patterns — no comments unless absolutely necessary |
| **Components** | Use shadcn/ui primitives where possible; prefer composition over inheritance |
| **State** | Use local state + hooks; no global state library |
| **Styling** | Use Tailwind CSS utility classes; custom CSS only for complex animations |
| **TypeScript** | Strict mode; avoid `any`; prefer `interface` over `type` for objects |
